# AI Study Buddy

# Volume H4 – Dashboard Experience

**Version:** 1.0 (Architecture Freeze)

---

# 1. Introduction

The Dashboard Experience defines the primary student interface after onboarding.

The Dashboard acts as the student's personal academic command centre throughout their university journey.

Its purpose is to transform complex academic information into a clear daily learning direction.

The Dashboard answers:

> "What should I learn, understand, and improve today?"

---

# 2. Dashboard Philosophy

The Dashboard follows these principles:

## Context Before Action

Students should first understand their current academic situation.

---

## Guidance Over Information

The Dashboard should recommend meaningful actions rather than display excessive data.

---

## Personalised Experience

Every student sees a different Dashboard based on:

- Semester.
- Subjects.
- Learning progress.
- Knowledge gaps.
- Study habits.

---

## Growth Visibility

The student should see evidence that their knowledge is growing.

---

# 3. Dashboard Structure

The Dashboard follows this information hierarchy:

```
Student Context

↓

Current Priorities

↓

Subject Progress

↓

Learning Actions

↓

Knowledge Growth

↓

BLIE Support
```

---

# 4. Dashboard Main Areas

The Dashboard consists of:

1. Academic Overview
2. Today's Learning
3. Subject Progress
4. BLIE Guidance
5. Knowledge Growth
6. Upcoming Activities
7. Recent Learning History

---

# 5. Academic Overview

Purpose:

Provide immediate understanding of the student's current academic position.

Displays:

- Student name.
- Current semester.
- Programme.
- Academic progress.
- Current learning stage.

Example:

"Semester 2 • Computer Science • 5 subjects"

The student immediately understands their current journey.

---

# 6. Today's Learning Section

Purpose:

Guide the student's daily study activity.

The system presents:

- Recommended study tasks.
- Upcoming topics.
- Revision suggestions.
- Preparation activities.

Examples:

"Prepare before tomorrow's Database Systems lecture"

"Review prerequisite concepts for Algorithms"

---

# 7. BLIE Daily Guidance

BLIE appears as a learning companion.

Possible interactions:

- "You have a lecture tomorrow. Would you like a preparation overview?"
- "You struggled with this concept previously. Let's review it."
- "Your exam is approaching. Here is a revision plan."

BLIE guidance is contextual, not random.

---

# 8. Subject Progress Overview

Purpose:

Provide a quick view of academic status.

Each subject displays:

- Progress.
- Current topic.
- Knowledge confidence.
- Upcoming activities.

Example:

```
Database Systems

Topic:
Relational Model

Status:
Learning

Next:
Practice Questions
```

---

# 9. Knowledge Growth Indicator

The Dashboard provides a simplified view of PLKG growth.

It should communicate:

- Knowledge acquired.
- Topics connected.
- Learning progress.

The Dashboard does not replace the full PLKG view.

It provides motivation and awareness.

---

# 10. Learning Readiness

A key differentiator of AI Study Buddy is preparation before learning.

The Dashboard may show:

"Upcoming Topic"

↓

"Required Background Knowledge"

↓

"Preparation Status"

Example:

Before studying Machine Learning:

"Review:
- Linear Algebra basics
- Probability concepts"

---

# 11. Upcoming Academic Activities

Displays:

- Lectures.
- Assignments.
- Tests.
- Revision goals.

The purpose is academic awareness.

---

# 12. Recent Learning History

Students can see:

- Recently studied topics.
- BLIE conversations.
- Completed activities.
- Knowledge updates.

This reinforces continuity.

---

# 13. Empty States

The Dashboard handles early student states.

Examples:

## New Student

Shows:

"Your learning journey begins here."

---

## No Materials

Shows:

"Upload your first lecture note to start building your knowledge."

---

## No Activity

Shows:

"Ask BLIE about your first subject."

---

# 14. Offline Dashboard Experience

When offline, the Dashboard displays:

Available:

- Cached subjects.
- Downloaded materials.
- Previous learning progress.

Unavailable:

- Real-time AI processing.
- Cloud updates.

The student should understand the current state.

---

# 15. Dashboard Personalisation

The Dashboard evolves with the student.

First Semester:

Focus:

- Understanding subjects.
- Building study habits.

Middle Years:

Focus:

- Knowledge connections.
- Deeper learning.

Final Year:

Focus:

- Advanced knowledge.
- Projects.
- Graduation preparation.

---

# 16. Dashboard Interaction Principles

The Dashboard should:

## Reduce Anxiety

Students should know what to do next.

## Encourage Progress

Students should see improvement.

## Support Independence

Students remain responsible for learning.

## Enable Discovery

Students discover useful learning paths naturally.

---

# 17. Architecture Freeze

The Dashboard Experience establishes the student's daily interaction centre.

The Dashboard shall:

1. Present academic context clearly.
2. Guide daily learning activities.
3. Connect students with BLIE.
4. Reflect PLKG growth.
5. Support preparation before learning.
6. Adapt throughout the student's university journey.

The Dashboard is the primary bridge between the student and the AI Study Buddy ecosystem.
